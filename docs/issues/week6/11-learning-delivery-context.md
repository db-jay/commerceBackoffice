## 목적
- 주니어가 `delivery 상태 전이와 출고 흐름 이해`를 구현 전에 설명할 수 있게 만든다.

## 숙지해야 할 내용
- 주문(order)과 배송(delivery)은 왜 같은 테이블/책임으로 다루지 않는가
- 배송 상태 전이는 왜 단순 문자열 변경이 아니라 운영 단계 의미를 가져야 하는가
- 송장 번호는 왜 아무 시점에나 넣으면 안 되고 특정 상태와 연결되어야 하는가
- 출고 이후 배송 흐름을 분리하면 claim/settlement 단계에서 무엇이 쉬워지는가
- 잘못된 상태 전이를 막지 않으면 운영 데이터에서 어떤 혼선이 생기는가
- W6가 이후 claim / supplier / settlement 학습과 어떻게 이어지는가

## 스스로 설명해보기
- `order`와 `delivery`를 나누면 어떤 변경이 쉬워지고 어떤 연결 포인트가 생기는가?
- 출고 준비 전인데 송장 번호를 먼저 저장하면 왜 문제가 될 수 있는가?
- `READY -> SHIPPED -> IN_DELIVERY -> DELIVERED` 같은 상태 전이는 운영 관점에서 각각 무엇을 의미하는가?
- 배송 상태를 막 바꿀 수 있게 두면 반품/교환/정산 단계에서 어떤 문제가 커지는가?
